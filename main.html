MyStop Case Study

User Research

2025

Preview

Pavithra

Documentation

Examining usability challenges in the MyStop transit app and creating an improved design solution for the State College community. UX Research, Design, Usability Testing.

üîçProblem Statement

The MyStop app, intended to facilitate seamless public transit experiences in State College, presents usability issues such as complex navigation, inaccurate real-time data, and disruptions due to external site redirections. These challenges diminish user satisfaction and trust, necessitating a comprehensive redesign.

‚úèÔ∏èResearch Focus

  • How do users currently interact with the MyStop app?
  • What are the most significant pain points in the user experience?
  • How can we redesign the app to improve navigation and trip planning?

üß≠Research Methods

  • Online surveys of current app users
  • In-depth interviews with regular commuters
  • Usability testing of the current app
  • Competitive analysis of other transit apps

üë•Key Findings

  • 62% of users reported difficulties navigating the app
  • 55.6% rarely check real-time updates due to accuracy concerns
  • 61% expressed dissatisfaction with the trip planning feature
  • Users frequently abandon the app when redirected to external sites

üí¨User Insights

Navigation Issues

  • "The direction feature is confusing, it's a struggle to find which is the destination and which is the start point."
  • "Every time I use the 'current location' feature, it directs me to a bus stop that's way farther than the one I usually use."

External Redirects

  • "When planning a trip, mystop sends you to google maps and google maps doesn't have the right bus routes."
  • "I wanted directions for the bus, but it gave me walking directions instead! Completely useless."

üë§User Persona

Alex, 21 - Penn State Student

  • Final-year undergraduate student with no car
  • Depends entirely on public transportation for daily commutes
  • Uses MyStop app to get to classes, grocery stores, and meet friends

Pain Points

  • Inconsistent bus information on Sundays and off-peak hours
  • Issues with NV routes information being inaccurate or unclear
  • App redirects to Google Maps, which doesn't provide bus-specific information
  • App suggests stops far from his actual location

üí°Solution

1. Integrated Trip Planning

Built in-app trip planning that keeps users within the app, eliminating external redirects and ensuring bus-specific information is always available.

2. Enhanced Real-Time Tracking

Improved data accuracy with focus on weekend and off-peak hours when service is often less predictable and reliable.

3. Simplified Navigation

Redesigned information architecture with clear visual hierarchy and intuitive user flow to reduce cognitive load.

üß±User Flow Redesign

Map Homescreen

Shows real-time bus locations and nearby stops, with a clean interface that prioritizes the most relevant information for the user's current context.

Trip Planner

Simple form for entering start and destination points, with smart location detection and suggestions based on commonly used stops.

Route Selection

Displays options with travel times, walking distances to stops, and expected arrival times, allowing users to choose based on their preferences.

Interactive Map View

Shows real-time updates and turn-by-turn directions within the app, with clear visual indications of bus routes and stops.

üé®Design Process

Research & Analysis

  • Conducted surveys and interviews to understand user needs
  • Created journey maps to identify pain points
  • Analyzed competitor transit apps for best practices

Information Architecture

  • Reorganized app structure for intuitive navigation
  • Created simplified user flows
  • Developed clear visual hierarchy

Wireframing & Prototyping

  • Created low-fidelity sketches to test concepts
  • Developed interactive prototypes
  • Conducted usability testing with target users

High-Fidelity Design

  • Designed UI with focus on accessibility
  • Created consistent design system
  • Optimized for both new and returning users

üìäResults & Impact

User Experience Improvements

  • Integrated trip planning keeps users within the app
  • More accurate real-time data, especially during off-peak hours
  • Clear visual representation of bus routes and stops
  • Streamlined navigation with fewer steps to complete key tasks

Testing Results

  • 87% of test users preferred the new design over the original app
  • Task completion time reduced by 40%
  • Error rate decreased by 65%
  • User satisfaction scores increased from 2.3 to 4.6 on a 5-point scale

üèÅConclusion

Key Achievements

The primary intent of the MyStop app redesign was to simplify the user experience and resolve the significant usability challenges identified through user research. The redesign focuses on creating a more intuitive and seamless experience for State College commuters, making it easier to find accurate bus routes without external distractions.

Design Principles

Every design decision was made with the users in mind, offering a clear and consistent structure, reliable real-time updates, and an overall reduction in cognitive load. The intent was to create an app that feels intuitive for both first-time and regular users, ultimately transforming MyStop into a more dependable and user-friendly transit tool for the State College community.

This user experience research and redesign project aims to improve public transit accessibility and usability for the State College community.

BudgetGuru: Financial App for Students

User Research & Interface

2025

Preview

Pavithra

Prototype

Creating a financial management platform tailored for student needs, addressing budgeting, savings, and loan repayment challenges. UX Research, User Testing, Mobile App Design.

üîçProblem Statement

Students struggle with financial management, often lacking effective tools to track expenses, manage loans, and develop healthy financial habits. Existing solutions are either too complex, not tailored for student life, or require excessive manual input, which leads to low engagement and frustration.

‚úèÔ∏èResearch Goals

  • Study apps, tools, and methods students currently use for financial management
  • Understand key challenges in budgeting, saving, and loan repayment
  • Explore factors that motivate or discourage consistent financial tracking
  • Identify gaps in current financial management solutions for students

üß≠Research Methodology

  • Survey of 14 international graduate and PhD students
  • In-depth interviews with 10 students from diverse backgrounds
  • Competitive analysis of existing financial apps
  • User testing of prototype designs

üë•Key Research Findings

  • 50% of respondents had student loans, but 55.6% rarely or never checked balances
  • 71.4% wanted expense tracking and savings goals features
  • 50% of students didn't track their spending at all
  • Only 14.3% used spreadsheets and just one person used a budgeting app

üéØUser Personas

Shreya (25, Business Major)

Background: Aspiring consultant who is financially aware but struggles with impulse spending on shopping, dining out, and travel. Has $45,000 in student loans.

Pain Points: Overspends on non-essentials, finds loan management complicated, needs budgeting insights without feeling restricted, prefers user-friendly tools with personalization.

Rishi (27, Computer Science)

Background: Graduate student from India with $60,000 in education loans. Between academic commitments and research, financial management takes a backseat.

Pain Points: Struggles tracking expenses and loan repayments, difficulty prioritizing payments, wants to understand interest accumulation, prefers automation over manual tracking.

üëÅÔ∏èStudent Financial Behaviors

Tracking Habits

"I tried using spreadsheets, but I always forget to update them. After a few weeks, I just give up."

Loan Awareness

"I honestly don't know how much interest is accumulating on my loans each month. I'm too scared to check."

Budget Challenges

"I make budgets all the time, but sticking to them is impossible. I need something that keeps me accountable without making me feel restricted."

Financial Goals

"I want to save for travel and my future career development, but I don't have a clear plan for how to balance that with my student loan payments."

üí°BudgetGuru Solution

Core Features

  • Expense Tracking & Budget Alerts - Help students see where their money goes
  • Loan Repayment Dashboard - Visualizes debt, interest, and repayment progress
  • Savings Goals - Track progress toward financial milestones
  • Gamification & Rewards - Motivate better financial habits
  • Automated Payment Reminders - Prevent missed payments
  • Financial Literacy Resources - Educational content on money management

Key Differentiators

  • Designed specifically for student financial needs
  • Focus on loan management and repayment strategies
  • Simplified interface requiring minimal financial knowledge
  • Personalized insights based on spending patterns
  • Gamification elements to encourage consistent use

üì±User Interface Design

Intuitive Onboarding

Simple setup process that quickly captures financial information without overwhelming the user

Financial Dashboard

At-a-glance view of budget status, upcoming payments, and savings progress

Expense Categories

Visual breakdowns of spending by category with trend analysis over time

Loan Simulator

Interactive tools to visualize different repayment strategies and their long-term impact

Goal Tracking

Visual representation of progress toward saving targets with celebration of milestones

Reward System

Achievement badges and point system to encourage consistent financial management

üíªDesign Process

Ideation

Conducted a 30-minute brainstorming session to generate feature ideas and prioritized them using dot-voting. Created user journey maps to understand pain points and opportunities.

Information Architecture

Developed a structured flow from onboarding through core functionality, ensuring intuitive navigation and minimal cognitive load for users.

Wireframing

Created low-fidelity sketches to explore navigation, input methods, and visualization approaches for financial data.

Prototyping & Testing

Developed interactive prototypes and tested with target users, iterating based on feedback to refine the experience.

üîÑCore User Flow

1. Onboarding

  • Quick and intuitive setup where users enter loan details
  • Define budgeting goals and set financial preferences
  • Connect bank accounts for automatic transaction tracking

2. Dashboard

  • Centralized hub providing real-time financial overview
  • Snapshot of budget status and loan repayment progress
  • Quick access to all main features from one screen

3. Expense Tracking

  • Automatic categorization of transactions
  • Visual trends and spending insights
  • Budget alerts when approaching category limits

4. Loan Management

  • Repayment simulator with different strategy options
  • Interest calculation and visualization
  • Payment schedule and automation features

5. Savings Goals

  • Personalized milestone creation
  • Progress tracking with visual feedback
  • Automated allocations from spending categories

Expected Impact

  • Increased financial awareness among college students
  • Improved loan repayment consistency
  • Reduction in financial stress and anxiety
  • Development of long-term positive financial habits

Future Development

  • Integration with university financial aid systems
  • Social features for roommate expense sharing
  • Advanced financial forecasting tools
  • Expanded financial education resources

Implementation Strategy

  • Beta testing with student focus groups
  • Partnership with university financial services
  • Gradual feature rollout based on user engagement
